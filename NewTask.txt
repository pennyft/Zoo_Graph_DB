Задача: Графовая база данных "Музыкальный фестиваль"

Описание предметной области:
На музыкальном фестивале выступают различные исполнители и группы.
Каждый исполнитель выступает на определённой сцене в определённое время.
Исполнители играют в разных музыкальных жанрах.
Между исполнителями существуют творческие связи (коллаборации).
За каждой сценой закреплены технические сотрудники.

Таблицы узлов (NODE):
1. Artist  — исполнители/группы (имя, жанр, страна, год_основания)
2. Stage   — сцены фестиваля (название, вместимость, тип: main/small/acoustic)
3. Genre   — музыкальные жанры (название, описание)
4. Crew    — технический персонал (имя, должность, опыт)

Таблицы рёбер (EDGE):
1. PerformsAt  — исполнитель выступает на сцене
               (Artist → Stage)
               атрибуты: время_выступления, длительность_мин, гонорар

2. PlaysIn     — исполнитель играет в жанре
               (Artist → Genre)
               атрибуты: основной_жанр (BIT), с_какого_года

3. CollabWith  — творческое сотрудничество между исполнителями
               (Artist → Artist)
               атрибуты: тип (запись/концерт/тур), год, статус

4. Manages     — сотрудник обслуживает сцену
               (Crew → Stage)
               атрибуты: с_какой_даты, роль, смена

Примеры запросов MATCH:
1. Найти всех исполнителей одного жанра на одной сцене
   Цепочка: Genre ← (PlaysIn) ← Artist → (PerformsAt) → Stage

2. Найти технический персонал сцен где выступают
   исполнители определённого жанра
   Цепочка: Genre ← (PlaysIn) ← Artist → (PerformsAt) → Stage ← (Manages) ← Crew

3. Найти исполнителей которые сотрудничали с теми
   кто выступает на главной сцене
   Цепочка: Artist → (CollabWith) → Artist → (PerformsAt) → Stage

Примеры запросов SHORTEST_PATH:
1. Найти цепочку творческих связей между двумя исполнителями
   Шаблон "+"

2. Найти все коллаборации глубиной до 3 шагов
   Шаблон "{1,3}"
